# Инструкция по запуску сервиса генерации документов 152-ФЗ

## 🚀 Быстрый старт

### 1. Запуск через Docker Compose (рекомендуется)

```bash
# Сборка и запуск всех сервисов
docker compose up -d --build

# Проверка статуса
docker compose ps

# Просмотр логов
docker compose logs -f
```

Сервис будет доступен по адресу: **http://localhost:3000**

### 2. Локальная разработка

#### Бэкенд:
```bash
cd backend
npm install
npx prisma generate
npx prisma migrate dev
npm run dev
```

Бэкенд запустится на **http://localhost:4000**

#### Фронтенд:
```bash
cd frontend
npm install
npm run dev
```

Фронтенд запустится на **http://localhost:5173** (Vite dev server)

## 🔐 Первый вход

1. Откройте http://localhost:3000
2. Нажмите "Зарегистрироваться"
3. Введите email и пароль (минимум 6 символов)
4. После регистрации войдите с теми же данными

---

## 📝 Генерация документов

1. Выберите нужные документы из каталога (29 документов по 6 категориям)
2. Заполните форму организации:
   - Наименование организации
   - ИНН (10 или 12 цифр)
   - Юридический адрес
   - ФИО ответственного за ПДн
   - Должность ответственного
   - Дата документа
3. Нажмите "Сгенерировать"
4. ZIP-архив с документами автоматически скачается

---

## 🔍 Проверка версий

При запуске фронтенд автоматически проверяет версию сервера через `/api/health`.

Если версии не совпадают:
- Отображается предупреждение в UI
- Блокируется возможность генерации документов
- Показываются ожидаемые и фактические версии

**Ожидаемые версии:**
- Server Version: `1.0.0`
- API Version: `v1`

## 🛠️ Переменные окружения

Скопируйте `.env.example` в `.env` в папке `backend`:

```bash
cp backend/.env.example backend/.env
```

Основные переменные:
- `DATABASE_URL` - подключение к PostgreSQL
- `JWT_SECRET` - секретный ключ для JWT (измените на свой!)
- `PORT` - порт бэкенда (4000)

## ⚠️ Важные замечания

1. **Безопасность**: Измените `JWT_SECRET` в production!
2. **Память**: Временные файлы автоматически очищаются после отдачи ZIP
3. **Юридическая информация**: Сервис предоставляет шаблоны. Рекомендуется консультация с юристом.
4. **Версионность**: При обновлении проверяйте совместимость версий клиента и сервера

## 🆘 Решение проблем

### Ошибка подключения к БД:
```bash
docker compose restart db
docker compose logs db
```

### Ошибка версий:
- Обновите бэкенд до версии `1.0.0`
- Пересоберите образы: `docker compose up -d --build --force-recreate`

### Ошибки в логах:
```bash
docker compose logs backend
docker compose logs frontend
```
